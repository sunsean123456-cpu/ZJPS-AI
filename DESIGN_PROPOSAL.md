# 绿色评价 Web 客户端 - 多模型并行分析设计方案

## 📋 参考：Helixa 的核心逻辑

Helixa 演示展示了一个高效的多模型查询模式：
1. **并行查询** - 同时调用多个 AI 模型（Helixa、ChatGPT-4o-mini、DeepSeek）
2. **并排展示** - 三个模型的回答以列式布局并行显示
3. **合并响应** - 一键将多个回答综合成一份统一报告

---

## 🎯 在绿色建筑评价中的应用

### 场景 1：技术方案评审
用户输入："分析这个项目的节能技术方案"

### 场景 2：材料合规性检查
用户上传材料后，并行检查：
- ✅ **完整性检查** - 是否缺少必要文件
- ✅ **真实性检查** - 数据逻辑一致性

评审完成后，并行生成多维度建议：
- 🔧 **技术改进** - 如何提升得分
- 📊 **材料优化** - 补充哪些证明材料
- 💡 **创新建议** - 可申报的创新点

## 🔧 技术实现方案

### 前端组件结构

```tsx
// 新增组件
<MultModelResponse>
  <ModelCard model="building-code" status="complete" />
  <ModelCard model="carbon-calc" status="complete" />
  <ModelCard model="economic" status="loading" />
  <MergeButton onClick={handleMerge} />
</MultModelResponse>

// 合并后的报告
<MergedReport>
  <ReportSection title="总体评价" />
  <ReportSection title="各维度得分" />
  <ReportSection title="改进建议" />
</MergedReport>
```

### 后端 API 设计

```python
# 并行查询多个模型
POST /api/evaluation/multi-analysis
{
    "project_id": 123,
    "query": "分析节能技术方案",
    "models": ["building-code", "carbon-calc", "economic"]
}

# 流式返回每个模型的结果
{
    "type": "model_result",
    "model": "building-code",
    "status": "complete",
    "result": {...}
}

# 合并请求
POST /api/evaluation/merge-analysis
{
    "project_id": 123,
    "analysis_ids": ["bc_xxx", "cc_xxx", "ec_xxx"]
}
```

## 📱 交互流程

### 流程 1：技术方案分析

```
1. 用户输入："分析这个项目的节能技术方案"
      ↓
2. 前端显示：🔄 正在查询 3 个分析模型...
      ↓
3. 并行调用后端 API（流式返回）
      ↓
4. 每个模型完成后，对应卡片显示 ✅
      ↓
5. 全部完成后，显示 [🔄 合并分析结果] 按钮
      ↓
6. 用户点击合并 → 生成综合报告
      ↓
7. 报告可展开/折叠各模型详情
```

### 流程 2：材料预审

```
1. 用户上传材料
      ↓
2. 自动触发并行检查：
   - 完整性检查
   - 真实性检查  
   - 合规性检查
      ↓
3. 并排显示三个检查结果
      ↓
4. 合并显示：✅ 可以启动评审 / ❌ 需要补充材料
```

## 💡 关键设计决策

### 1. 模型选择
- **方案 A**：调用不同的专业模型（如 GPT-4、Claude、本地模型）
- **方案 B**：同一模型，不同的 prompt 角色（规范专家、碳排专家、经济专家）
- **推荐**：方案 B（成本更低，响应更快，效果可控）

### 2. 合并策略
- **方案 A**：简单拼接各模型结果
- **方案 B**：用 AI 模型综合各模型结果生成报告
- **推荐**：方案 B（更智能，但需要额外一次 API 调用）

### 3. 流式 vs 等待全部
- **方案 A**：等所有模型完成后一次性显示
- **方案 B**：流式显示，每个模型完成即显示
- **推荐**：方案 B（用户体验更好，减少等待感）
